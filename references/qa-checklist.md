# Step 8 · 每页 QA 自检清单

## 使用方式

**在 Step 7 一次性生成期间,每生成完一页就立即跑一遍这个清单**,发现问题当场修正,不要堆到最后。等 30 页都生成完才发现漂移,修正成本是单页的 N 倍。

引擎(deck_engine.js)+ 令牌(themes.json)已经把"字号层级、颜色、对齐、页脚格式"锁死了,所以 QA 重心从"排版对不对"转移到"**用法对不对、内容好不好**"。

---

## 8 项核心检查

### 1. spec-lock 一致性(最重要)
- [ ] 这页用的版式 = spec-lock 页面映射表里的版式?
- [ ] theme_id 没变?没有在版式函数之外自己 addText/addShape 塞私货?
- [ ] 没有出现 themes.json 之外的颜色(生成代码里 grep 十六进制字面量,只允许出现在 themes.json)?

### 2. 容量限制
- [ ] contentRows ≤ 5 行、threeCards 2-4 卡、timeline 3-6 节点、processSteps 3-5 步、table ≤ 8 行、kpiGrid 2-4 个?
- [ ] 超限的处理方式是"拆页/换版式",而不是塞进去?

### 3. 文字密度(按 brief 的"用途")
| 用途 | 单页字数上限(中文) |
|---|---|
| 现场演讲 | ≤ 80 字 |
| 线上会议 | ≤ 150 字 |
| 自看材料 | ≤ 300 字 |
- [ ] 没有超过 3 行的段落?列表项 ≤ 6 个?
- [ ] 标题是"这页要观众记住的话",不是"本页内容的分类名"?

### 4. 标题措辞一致性
- [ ] 全 deck 标题风格统一(都用陈述句 / 都用名词短语,不混)?
- [ ] kicker 格式统一(如都是"中文 · ENGLISH")?
- [ ] 标点统一(中文页用中文标点)?

### 5. 明暗节奏
- [ ] 对照 spec-lock 的节奏检查行:hero 页位置正确?
- [ ] 没有连续 3 页以上纯内容页?没有相邻两个 hero 页(封面+目录除外)?

### 6. 数据与图表
- [ ] 所有 table 都填了 `source`?所有 imageText 都有 `caption`?
- [ ] bigStat 的柱状相对高度与真实数据比例一致(不夸大)?
- [ ] 同一数据在不同页数值一致(P 值、百分比)?
- [ ] kpiGrid 的 good/dir 语义正确(流失率下降是 good:true + dir:"down")?

### 7. 深色/高饱和主题专项(dark 主题与 swiss 系必查)
- [ ] hero 页副文字用的是 heroT2,没有手动指定更暗的颜色?
- [ ] 符号只用已验证可渲染的(✓ ✗ → ↑ ↓),不用 emoji(❌✅ 可能缺字形)?
- [ ] 若需打印讲义:确认没选深色主题(否则回退浅色主题重生成)?

### 8. 演讲者备注与占位符
- [ ] brief 要求备注时,每页 `notes` 都填了(是口播稿,不是页面文字的复读)?
- [ ] 没有遗留 "[ ]"、"TBD"、"占位"、"placeholder" 文字?imageText 的 placeholder 只在等用户图片时允许存在,交付前必须确认?

```markdown
## QA Summary

- 总页数:X · 主题:xxx · 引擎版式使用:cover×1, contentRows×4, ...
- 自检通过:X / X 页
- 修正记录:
  - P05:contentRows 6 行 → 拆成两页
  - P09:表格缺 source → 已补
- 待用户拍板的开放项:
  - P07 引用文献年份待确认
  - P09 的占位图等用户提供
```
